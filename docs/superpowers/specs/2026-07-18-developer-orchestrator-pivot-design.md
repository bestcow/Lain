# lain 개발자 오케스트레이터 전향 — 설계 (2026-07-18)

> 배경: 컴패니언 기능(플래너·음성 대화·화면 감시·3D 캐릭터)이 누적되며 제품 컨셉이 흐려짐.
> 사용자 결정: **철저하게 개발자를 위한 오케스트레이터로 전향.** 정리(다이어트)와 강화(CC 관제탑·루프 엔지니어링)를 병행한다.

## 1. 컨셉 재정립 (P1)

한 줄 정의: **"여러 레포를 가진 개발자의 로컬 AI 오케스트레이터 — Claude Code 작업을 지휘·검증·병합하는 관제탑, 얼굴은 레인."**

- PLAN.md §1(목적)·§2(목표/비목표) 개정. 비목표에 추가: "일반 비서·일정관리·컴패니언 기능(캘린더, 리마인드, 범용 화면 감시, 음성 대화)은 만들지 않는다. 페르소나는 인터페이스이지 제품이 아니다."
- CLAUDE.md 첫 줄을 같은 정의로 갱신. 이후 기능 추가 제안은 이 문장이 게이트.
- 공개 README·레포 설명은 **보류** — 이번 작업에서 건드리지 않는다.

## 2. 다이어트 (P2)

### 2.1 플래너 제거

- 제거: `src/main/planner.ts`, `src/shared/planmath.ts`, `PlannerPanel.tsx`, `/plan` 슬래시·📅 진입점, IPC 9종(`planner:*`)+`planner:updated` 브로드캐스트(ipc.ts·preload·types.ts 3곳), 레인 도구 4종(`plan_manage`/`plan_view`/`plan_tag_manage`/`plan_section_manage`), SYSTEM_PROMPT 플래너 규칙, scheduler 리마인드 틱, 텔레그램 `/plan`+리마인드 푸시 버튼, 브리핑 포함 로직, PrefsModal '플래너' 섹션(설정 16키).
- 보존: DB 테이블 `plan_items`/`plan_tags`/`plan_sections`와 기존 데이터(마이그레이션·DROP 없음 — SQLite 잔존 테이블은 무해). store.ts의 테이블 생성 코드는 제거.
- 방어: 설정 저널에 남은 플래너 키는 무시되도록(알 수 없는 키 무해 원칙 확인).
- 관련 테스트(planmath·planner·리마인드) 제거, 전체 스위트 그린 유지.

### 2.2 음성 정리

- **PC 음성입력 숨김**: 입력바 마이크 PTT·녹음 UI를 설정(기본 off) 뒤로. STT 경로(`voice:stt`·Groq·stt-filter·무음 게이트) 코드 보존.
- **TTS 유지**: 엔진 3종(Edge/Supertonic/GPT-SoVITS)·한글화·속도 슬라이더 현행 유지.
- **디스코드 유지·동결**: 코드 변경 0. HANDOFF의 음성 T6~T10 항목을 "동결(개발 안 함)"로 표기만 갱신.

### 2.3 3D 캐릭터/Rive 보류

- 코드 변경 0(코드에 미반영 트랙). RIVE-PLAN.md·CHARACTER-BIBLE은 문서로만 유지, HANDOFF 다음 작업 후보에서 제외.

## 3. UI 재조준 — "캐릭터의 방" → "관제실" (P3)

- **캐릭터 컴팩트화**: 평소 작은 아바타+말풍선(LainBubble 유지)만. 발화·이벤트 때 부각(확대/강조). 상시 대형 일러스트 영역 폐지, 확보 공간은 관제 정보에.
- **프로젝트 카드 = 관제 1열**: 브랜치·미커밋/미푸시·진행 작업·대기 승인/결재·마지막 CC 활동을 카드에서 즉시 확인(기존 메타 세로 배치의 확장).
- **quips 재조준**: 트리거 15종을 개발 이벤트 중심으로 재편 — 작업 완료/실패·머지·검증 실패·레드 상태·장시간 작업 등. 비개발 트리거는 제거 또는 개발 문맥으로 교체. 말수 슬라이더·에스컬레이션 메커니즘은 유지.
- 상태 표식·다이제스트 용어를 개발 지표 중심으로 정리(기능 변경 없는 문구 정돈 포함).

## 4. 오버레이 — 개발 전용 재조준 (P4)

- **화이트리스트 캡처**: 포그라운드 앱이 지원 목록일 때만 캡처. 1차 지원: 터미널(Windows Terminal·conhost)·VS Code·브라우저의 localhost/GitHub 탭. 그 외 화면은 캡처 자체를 안 함 — 비용·프라이버시 동시 개선.
- **개발 맥락 프롬프트**: 에러 메시지·스택트레이스·빌드 실패·테스트 실패 감지 시 짧은 조언 또는 "이거 내가 맡을까요?" → 수락 시 `start_task` 연결.
- **지원 환경 = 설정 데이터**: 지원 앱 목록을 설정으로 관리(하나씩 추가하며 확장, 코드 변경 없이 성장).
- 기존 토글(기본 off)·슬라이드 표시·적응형 시간은 유지.

## 5. 강화 A — CC 관제탑 (P5)

기존 ccsessions(list/read/ask fork)·cchooks(양방향 훅) 기반 심화. 순서 C1→C2→C3→C4.

- **C1 세션 대시보드**: 진행/최근 CC 세션을 레포 카드에 표시(세션 제목·상태·마지막 활동 시각).
- **C2 이어받기(승격)**: CC에서 하다 만 작업을 레인 작업으로 승격 — 세션 요약 → 핸드오프 md 생성 → worktree Navi 재개. 기존 ask-fork의 확장.
- **C3 컨텍스트 왕복 심화**: CC 세션 시작 시 다이제스트 주입(기존) + 세션 종료 시 결과 요약을 레인 다이제스트·학습에 역반영.
- **C4 활동 타임라인**: 레포별 CC 세션 이력 뷰(훅 이벤트 기반).

## 6. 강화 B — 루프 엔지니어링 (P6)

작업 사이클(지시→작업→검증→리뷰→병합)의 구멍 메우기. 착수 순서 L2→L1→L5→L6→L3→L4.
컨벤션 준수: 판단(요약·심사)은 LLM, 저장·주입·집계·상태 전이는 L0 결정론.

- **L2 실패 회고 메모**: verify/review 실패 시 원인 한 줄 요약(LLM) → 프로젝트 학습으로 자동 저장(L0) → 재시도 프롬프트에 주입. 같은 원인 반복 실패 차단. 기존 lessons 배관 재사용.
- **L1 독립 심사관**: done 판정 전 별도 검증 세션이 diff와 완료 조건을 대조 판정. 작업 Navi의 자기 보고를 신뢰하지 않는 구조. verify_cmd(명령 실행)와 상보 — 명령이 못 잡는 "요구와 다른 구현·부분 완료"를 잡는다.
- **L5 지적사항 자동 재작업**: 리뷰 리젝 시 지적사항을 수정 지시로 변환(L0) → 같은 worktree에서 이어 수정 → 재리뷰. 무한 루프 방지: 재작업 횟수 상한(기본 2회), 초과 시 사람에게 에스컬레이션.
- **L6 성적표(루프 관측성)**: 1회 통과율·재시도율·리젝 사유 top-N을 SQLite 집계(L0)로 주간 다이제스트에 포함. task_events 기반, LLM 비용 0.
- **L3 완료 조건 체크리스트(DoD 정형화)**: clarify 단계가 완료 조건을 체크리스트로 정형화해 구조화 저장(L0), verify·review·L1 심사관이 항목별 O/X 판정. PLAN §21.3 elicitation의 첫 슬라이스.
- **L4 리뷰 세기 다이얼**: 작업별 리뷰 모드 — 경량/표준/적대(독립 스켑틱 N표). 적대 모드는 토큰 비용이 커 opt-in. 수동으로 돌리던 적대 리뷰 워크플로의 파이프라인 옵션화.

## 7. 실행 순서·검증

**P1** 컨셉 문서 개정 → **P2** 다이어트(2.1 플래너 → 2.2 음성 → 2.3 표기) → **P3** UI 관제실 → **P4** 오버레이 재조준 → **P5** CC 관제탑(C1→C2→C3→C4) → **P6** 루프(L2→L1→L5→L6→L3→L4).

- 각 단계마다 `npm run typecheck` + vitest 전체 그린 확인 후 다음 단계.
- 페이즈 완결 시 `npm run deploy`로 설치본 반영.
- 라이브 검증 필요(자동검증 불가): 오버레이 화이트리스트 실캡처 동작·quips 재편 후 실발화·C1 대시보드 실표시·C2 이어받기 1건.
- 페이즈별 변경 요약을 한 줄로 기록.

## 8. 리스크·완화

- **플래너 습관 소실**: 텔레그램 /plan·리마인드가 사라짐. 데이터 보존 + git 이력으로 복원 가능. 제거 커밋을 단독 커밋으로 분리해 revert 용이하게.
- **quips·오버레이 재조준**: 기존 라이브 검증 미완 항목 위의 변경 — 재조준 후 라이브 검증 1회 필수(§7).
- **설정 키 제거 호환**: 기존 설치본 설정 저널의 잔존 키가 무시되는지 확인(부팅 회귀 방지).
- **L5 재작업 루프 폭주**: 횟수 상한+에스컬레이션으로 방어(§6).
- **L1/L4 토큰 비용 증가**: L1은 diff+DoD만 읽는 경량 세션으로 설계, L4 적대 모드는 opt-in.
