# Changelog

Lain의 공개 릴리스 변경 기록. 형식은 [Keep a Changelog](https://keepachangelog.com/ko/1.1.0/)를 따르고, 버전은 [SemVer](https://semver.org/lang/ko/)를 쓴다.

## [1.2.0] - 2026-07-21

lain의 성격이 분명해진 판입니다. 일정·잡담 같은 컴패니언 기능을 걷어내고 **여러 레포를 가진 개발자의 작업 관제탑**으로 방향을 좁혔고, 처음 켠 사람이 뭘 해야 할지 알 수 있게 다듬었습니다. 이 버전부터 **개발이 공개 저장소에서 직접 이뤄집니다**.

### 새로 생김

- **처음 켰을 때 안내** — 등록된 프로젝트가 없으면 레인이 먼저 프로젝트 폴더 위치를 묻고, 답을 주면 그 자리에서 설정·스캔해 몇 개를 찾았는지 알려줍니다. 설정 창을 뒤질 필요가 없습니다.
- **개발자 관제탑으로 전향** — 작업 위임·검증·병합 흐름을 중심에 두고 화면을 재편했습니다. 승인 대기 배지, 작업 카드의 진행 상황, 실패한 작업의 회고·재시도(rework)·성적표가 들어왔습니다.
- **클로드코드 세션 연동** — lain 밖에서 돌린 클로드코드 세션을 목록에서 열어보고, 이어받아 계속 시킬 수 있습니다.
- **텔레그램에서 작업 조종** — 진행 중인 작업 카드가 실시간으로 갱신되고, 끼어들기·중지·변경사항(diff) 보기·검증 재실행을 버튼으로 처리합니다.
- **응답 속도 개선** — 레인이 말하는 즉시 화면에 나오고(스트리밍), 도구를 쓰지 않는 간단한 대화는 더 빠른 경로로 처리됩니다.
- **음성 답변 스트리밍**(PC) — 첫 문장부터 바로 읽어줍니다.
- **자동 백업** — 하루 한 번 대화·학습 데이터를 백업하고 오래된 것은 정리합니다.

### 좋아짐

- **작업량을 알아서 맞춤** — 인사 한 마디에 온 힘을 쓰던 것을 고쳐, 입력에 맞는 강도로 응답합니다.
- **입력창이 무슨 설정인지 보임** — 값만 덩그러니 있던 컨트롤에 `권한`·`모델`·`강도` 같은 이름표를 붙이고 레인 설정과 작업 설정을 구분선으로 나눴습니다.
- **프로젝트 아이콘 구분** — 이름이 비슷한 프로젝트가 똑같은 아이콘을 받던 것을 고쳐 색으로 구분되고, 엉뚱한 아이콘이 붙던 오매칭(예: `scowl`이 소 아이콘)도 없앴습니다.
- **"지금 뭐 하는 중"을 사람 말로** — 내부 용어 대신 읽으면 아는 문장으로 표시합니다.
- **화면이 가벼워짐** — 렌더러 번들 절반 축소(JS 1,039kB→502kB, CSS 101→65kB), 대화 열기 속도 개선(DB 인덱스 추가).
- **오버레이는 개발 화면에서만** — 에디터·터미널·개발 문서일 때만 관찰하고 그 밖에는 캡처 자체를 하지 않습니다.

### 고침

- **시크릿 파일 차단이 실제로는 동작하지 않던 문제**(최중대) — 자동 허용된 도구에는 검사기가 아예 호출되지 않아 `.env` 같은 파일이 그대로 읽힐 수 있었습니다. 차단을 항상 도는 경로로 옮겼고, 사용자의 전역 클로드 설정이 lain 세션에 섞여 들어가던 경로도 닫았습니다.
- **PC에서 답변 음성이 나오지 않던 문제** — 재생 배선이 빠져 있었습니다.
- **컴퓨터 시작 시 자동 실행이 조용히 꺼지던 문제** — 작업 스케줄러 방식으로 바꿔 근본 해결했습니다(12시간 동안 안 켜져 있던 사고).
- **정상 데이터를 손상으로 오판해 버리던 문제** — 손상 위치를 2차 판별해 멀쩡한 쪽을 보존합니다.
- 작업 취소 부활·인터럽트 유실·핸드오프 복원 누락 등 신뢰성 13건, 안정성·성능·UI 전수 감사 67건을 함께 반영했습니다.

### 바뀐 점 (사용하던 분들에게)

- **일정 관리(플래너) 기능이 제거됐습니다** — 개발 관제탑에 집중하기 위한 결정이며, 기존 데이터는 지워지지 않습니다.
- **PC 음성 입력이 숨겨졌습니다**(음성 답변은 유지).
- **작업 기록 파일이 바뀝니다** — `HANDOFF.md`·`UPDATE.md`는 이제 각자의 로컬 파일이라 저장소에 커밋되지 않습니다. 클론 후 `npm install`을 하면 틀(`*.example.md`)에서 자동으로 만들어지고, 앞으로 `git pull`이 충돌 없이 깔끔해집니다.
- 프로젝트 아이콘 색이 이번 판에서 한 번 바뀝니다(같은 테마를 쓰는 프로젝트를 구분하기 위함).

## [1.1.2] - 2026-07-13

안정성 패치 — 오래 켜두거나 일을 많이 시켜도 느려지거나 멈추지 않게 다듬었습니다.

### 고침

- **긴 응답을 받을수록 화면이 점점 무거워지던 문제** — 스트리밍 표시 방식을 개선해 응답 길이와 무관하게 가볍게 유지됩니다.
- **멈춘 응답을 강제 종료해도 내부 프로세스가 남던 문제** — 이제 실제로 정리되어 자원이 새지 않습니다.
- **내부 판정이 응답하지 않으면 새 작업이 시작되지 못하던 문제** — 60초 제한을 걸어 작업 흐름이 막히지 않습니다.
- **텔레그램 처리가 걸리면 수동 재시작 전까지 먹통이던 문제** — 스스로 감지하고 복구합니다.
- **성능 측정(벤치) 실행이 학습한 학습을 지우던 문제** — 측정 전 백업하고 끝나면 복원합니다(도중에 꺼져도 다음 부팅 때 복원).
- **화면 오류 시 자동 복구가 무한 반복될 수 있던 문제** — 점진적으로 간격을 늘려 재시도합니다.
- **클로드코드 연동이 드물게 사용자 설정 파일을 초기화할 수 있던 문제** — 기존 설정을 읽지 못하면 건드리지 않고, 쓰기도 안전하게 바꿨습니다.
- **컴퓨터 시작 시 자동 실행이 조용히 꺼지던 문제(Windows 11)** — 켜질 때마다 스스로 점검·복구합니다.
- 데이터베이스 손상 복구 경로에 회귀 테스트를 보강했습니다.

## [1.1.1] - 2026-07-11

### 고침

- **최신 Node에서 소스 빌드 설치가 실패하던 문제** — 디스코드 음성용 네이티브 모듈을 선택적 의존성으로 바꿔(순수 JS 폴백), 빌드 도구가 없는 Node 24+ 환경에서도 `npm install`이 정상 완료됩니다.
- **미출시 모델이 모델 선택지에 보이던 문제** — 아직 제공되지 않는 모델을 고르면 응답이 실패하던 것을 선택 목록에서 제외했습니다.
- **로컬 음성(Supertonic) 선택 시 발화마다 지연되던 문제** — 준비되지 않은 경우 20초를 기다리지 않고 즉시 기본 음성으로 대체합니다(`npm run setup:supertonic`으로 활성화).
- **오래 쓰면 부팅이 느려지던 문제** — 내부 기록 저널을 부팅 시 정리해(삭제된 대화·중복 제거) 무한히 커지지 않게 했습니다.

### 안전

- **위험 셸 명령 승인 게이트 확장** — 관리자(Lain)가 직접 실행하는 `git push`·`rm -rf`·의존성 설치·네트워크 다운로드 명령도 승인 버튼(PC·텔레그램)을 거칩니다(bypass 모드에서만 자동 통과). 기존엔 작업 Navi에만 적용됐습니다.

### 변경

- 설치 안내를 소스 빌드 우선으로 정리하고 앱·설치본 아이콘을 추가했습니다.
- 공개 저장소에서 내부 개발 문서를 제외했습니다.

## [1.1.0] - 2026-07-10

### 추가

- **플래너** — 통합 일정+체크리스트 캘린더가 추가됐습니다(월/주/일 전환). 태그·섹션으로 커스텀할 수 있고, 리마인드는 PC 알림과 텔레그램 버튼으로 옵니다. 방치된 할 일은 넛지로 알려주고, Lain에게 "내일 3시 미팅"처럼 말하면 바로 등록됩니다. 체크리스트 항목은 Lain에게 대신 처리해달라고 부탁할 수도 있습니다.
- **작업 오케스트레이션 확장** —
  - 작업 간 의존성: "A 끝나면 B 해줘"로 위임하면 선행 작업이 완료되는 순간 자동 착수합니다.
  - 같은 프로젝트 병렬: 설정 '프로젝트 병렬 cap'(기본 1)을 2~4로 올리면 한 레포에서 여러 작업이 동시에 돕니다.
  - 크로스레포 작업 그룹: 여러 레포에 걸친 한 기능을 묶어 위임하고, 전부 준비됐을 때만 일괄 병합합니다(중간 실패 시 병합분 자동 롤백).
  - 편집 되감기: Lain이 직접 고친 파일은 채팅의 diff 카드에서 턴 단위로 되돌릴 수 있고, 복원 직전 상태 카드로 '되돌리기 취소'도 됩니다.
  - 작업 큐·체크포인트·사용량 예산 가드·병합 rebase 폴백으로 작업 실행이 더 견고해졌습니다.
- **상호작용 대사** — 조작(감시 토글 등)에 레인이 즉각 짧게 반응하는 말풍선이 생겼습니다. 환경설정 '말수' 슬라이더(묵언~수다쟁이)로 조절하고, 묵언이면 관련 비용이 0입니다.
- **첫 실행 온보딩** — 처음 켜면 Claude 연결 확인 → 프로젝트 폴더 등록 → 호칭 설정을 안내하는 시작 위저드가 뜹니다(1회).
- **Codex 엔진(선택)** — 작업 Navi를 OpenAI Codex CLI로도 실행할 수 있습니다. Lain에게 "codex로 해줘"라고 지시하면 됩니다(별도 설치·로그인 필요, `npm install -g @openai/codex`). 관리자·판정은 Claude 그대로입니다.
- **클로드코드 연동 검증 완료** — 환경설정에서 켜면 Lain 밖에서 직접 실행한 `claude` 세션의 시작/종료를 Lain이 인지하고, 반대로 그 세션에는 Lain의 작업 현황이 자동 주입됩니다(양방향).
- **환경설정 확장** — API 키 직접 입력, 워크스페이스 스캔 폴더 설정, 데이터 백업/복원 안내가 추가됐습니다.

### 변경

- UI 조작 아이콘 40종을 일관된 아이콘 세트(reicon, MIT)로 교체했습니다.
- TTS 속도 슬라이더, 지난 대화와 새 내용 사이 경계선, 세션 새로고침 시각 표시 등 UX를 다듬었습니다.

### 안전

- **시스템 파괴 명령 승인 게이트** — PC 전원 끄기·디스크 포맷·레지스트리 삭제·루트 폴더 삭제 같은 명령은 Lain/Navi 어느 쪽이 시도하든 반드시 승인 버튼(PC·텔레그램)을 거칩니다. 일상 개발 명령(빌드·배포·node_modules 삭제 등)은 그대로 통과합니다.

### 고침 (학습·보고 신뢰성)

- **학습 정정이 완전해짐** — "그렇게 하지 마"라고 정정하면 Lain이 잘못 학습한 학습을 병합 파생본까지 즉시 철회합니다(retract_lessons). 큐레이터가 서로 모순되는 학습을 병합해 틀린 내용을 되살리던 문제도 차단.
- **이름을 밝히면 그 이름으로 부르던 오작동** — 턴 리뷰가 이름·신원 같은 정보를 행동 규칙으로 잘못 저장하지 않게 문턱을 강화했습니다(호칭 변경은 명시 지시가 있을 때만).
- **배포 허위 보고 제거** — 자기 배포(deploy_lain)가 실제로 시작됐는지 확인한 뒤 보고합니다. 조용히 실패하면 실패라고 말합니다.

### 고침 (오케스트레이터 견고성 — 독립 적대 리뷰 5건)

- **그룹 롤백 실패 시 복구 불능** — 크로스레포 그룹 롤백에서 되돌리기 자체가 실패하면(작업 폴더가 지저분한 경우 등) 병합 기록까지 지워 수동 복구가 불가능해지던 문제. 이제 기록을 보존해 언제든 다시 되돌릴 수 있습니다.
- **그룹 병합 도중 강제 종료 시 반쪽 병합 방치** — 여러 레포 일괄 병합 도중 앱이 죽으면 일부만 병합된 채 영영 남던 문제. 재시작 시 감지해 알려주고, 이어서 병합하거나 되돌리는 복구 경로를 제공합니다.
- **취소한 작업의 부활** — 작업에 끼어들기(인터럽트)를 건 직후 취소하면, 취소된 작업이 되살아나 이미 삭제된 작업 폴더에서 재개되던 경합 문제를 차단했습니다.
- **되돌리기 취소 불가** — 편집 되감기 확인창이 약속한 "복원 직전 상태로 다시 되돌리기"가 실제로는 화면 어디에도 없던 문제. 복원 후 채팅에 '↩ 복원 직전 상태' 카드가 남아 한 번 더 되돌릴 수 있습니다.
- **세션 새로고침 직후 옛 맥락 잔류** — 새로고침으로 기억을 비운 직후 첫 대화에 리셋 전 혼잣말 맥락이 새어 들어와 자기모순이 생기던 문제를 정리했습니다.

## [1.0.0] - 2026-07-04

첫 공식 릴리스.

### 주요 기능
